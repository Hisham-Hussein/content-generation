# Brand Validation Rules

Use this reference to decide whether normalization should continue, stop, or ask for clarification.

## Blocking Gaps

Stop when any of these are true:

- no usable tenant folder path was provided
- no identifiable tenant name can be determined
- no usable visual guidance exists at all
- no palette exists and there is no safe inference path
- no workable typography direction exists and there is no safe default path
- multiple brands or personas are present and no single active publishing brand can be resolved safely

## Non-Blocking Gaps

Continue when the missing information is useful but not fatal, and record it clearly:

- optional persona or audience notes are missing
- imagery references are incomplete but still directionally usable
- some render assets are absent but not required for version-one output
- extra raw materials are present but not relevant to the normalized contract

## Required Reporting

`brand-validation-report.md` should clearly state:

- what was found
- what was missing
- what was inferred
- what was defaulted
- whether any blocking issue remains
- whether the tenant is ready for draft normalization only or ready for approved-profile review

## Refresh Rule

- An existing approved normalized brand profile should not be refreshed automatically.
- Only rebuild or update the profile when the user explicitly asks.
