# Brand Material Intake

Use the tenant's original brand materials directly. Do not require a normalized profile for infographic generation.

## Read Directly From The Tenant Folder

Inspect what actually exists, including:

- `README.md`
- tenant `SKILL.md`
- brand-kit HTML files
- CSS or token files
- local brand assets
- approved examples or visual references

## Canonical Rule

For infographic generation, the canonical brand source is the tenant's original source material, not a flattened normalized profile.

## What Counts As A Real Blocker

Stop only if one of these is true:

- multiple brands, sub-brands, or personas conflict and the active one is not clear
- an essential render asset explicitly required by the tenant material is missing
- the source material is too contradictory to produce a defensible brand interpretation

## What Does Not Count As A Blocker

- no `normalized-brand-profile.md`
- no `brand-validation-report.md`
- no shared cross-tenant schema
- incomplete brand materials that are still directionally usable

## Quality Rule

- Prefer richer original materials over flattened summaries.
- Use the tenant's actual examples, assets, and source guidance when available.
- Preserve brand nuance instead of compressing everything into generic tone bullets.
